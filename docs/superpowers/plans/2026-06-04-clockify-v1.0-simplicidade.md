# Clockify v1.0 — Simplicidade para leigos (onboarding + default opcional + atividades aprendidas) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tornar o plugin super simples para usuários leigos — onboarding de 3 perguntas, atividade padrão (default) opcional, e uma única memória de "atividades aprendidas" (funde histórico + overrides) reconhecida pelo Claude — escondendo todo o encanamento técnico.

**Architecture:** Mantém a separação cérebro/IO. O código fica mais permissivo (o "trio" de defaults vira opcional, `load_defaults` nunca levanta) e mais enxuto (remove `from_event`, `suggest`, `add-override` e a seção `overrides`). Um módulo novo `learned.py` substitui `history.py` com um store em **lista** (`learned.json`): cada atividade tem `{match, project_name, task_name, tag_names, billable}`. O `add` grava (upsert por `match`) no sucesso; `learned add` grava por palavra-chave; `learned list` expõe tudo. **Todo o reconhecimento é feito pelo Claude** na conversa (não há matching no código). Precedência ao montar cada item: atividade aprendida > atividade padrão > perguntar.

**Tech Stack:** Python 3.12, argparse, httpx, pytest + respx, `uv`. Plugin Claude Code (`plugins/clockify-horas/`).

**Spec:** `docs/superpowers/specs/2026-06-04-clockify-v1.2-simplicidade-design.md` (o design foi nomeado "v1.2"; este esforço é **renomeado para v1.0** porque o plugin ainda não foi lançado — a versão permanece **`1.0.0`**, política pré-lançamento do `MAINTAINER.md`).

**Base:** repo em `master`. **Trabalhe num branch novo** `feat/v1.0-simplicidade` (não implemente em `master`). Todos os caminhos abaixo são relativos à raiz do repo; o projeto Python vive em `plugins/clockify-horas/` — rode `uv run pytest` de lá.

**Regra de dados (inviolável):** nenhum nome de cliente/projeto real nos testes, fixtures ou docs do repo. Use nomes demo (`Proj Demo`, `Tarefa Demo`, `Etiqueta Demo`, `Cliente Demo`). - [ ] **Step 3: Implementar `learned.py`**

Crie `src/clockify_horas/learned.py`:

```python
import json
import os
from pathlib import Path

from clockify_horas.config import config_root


def learned_path() -> Path:
    return config_root() / "learned.json"


def _norm(match: str) -> str:
    return match.strip().lower()


def read_learned(path: Path | None = None) -> list[dict]:
    """Lê a lista de atividades aprendidas. Ausente/corrompido/formato inesperado → []."""
    p = path or learned_path()
    if not p.exists():
        return []
    try:
        data = json.loads(p.read_text(encoding="utf-8"))
    except (OSError, json.JSONDecodeError):
        return []
    return data if isinstance(data, list) else []


def write_learned(data: list[dict], path: Path | None = None) -> Path:
    p = path or learned_path()
    p.parent.mkdir(parents=True, exist_ok=True)
    p.write_text(json.dumps(data, ensure_ascii=False, indent=2) + "\n", encoding="utf-8")
    if os.name == "posix":  # chmod 600 é POSIX-only
        p.chmod(0o600)
    return p


def record(
    match: str,
    project_name: str | None,
    task_name: str,
    tag_names: list[str],
    billable: bool,
    path: Path | None = None,
) -> None:
    """Upsert de uma atividade aprendida (dedup por `match` normalizado).

    Chaves espelham o item do JSON do `add` (project_name/task_name/tag_names/billable),
    para o Claude copiar direto sem renomear campo. `match` guarda o texto original.
    """
    entry = {
        "match": match,
        "project_name": project_name,
        "task_name": task_name,
        "tag_names": list(tag_names),
        "billable": bool(billable),
    }
    data = read_learned(path)
    for i, a in enumerate(data):
        if _norm(a.get("match", "")) == _norm(match):
            data[i] = entry
            break
    else:
        data.append(entry)
    write_learned(data, path)
```
